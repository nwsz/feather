Introduction

Feather docs

A whisper-light Chrome extension that watches Discord for new messages in real time — firing webhook alerts, logging everything to Supabase, and showing a clean overlay indicator right on the Discord page.

👁️
Live DetectionMutationObserver watches Discord's DOM. No polling, no API. New messages are caught the instant they appear.
🔔
Webhook AlertsPosts to a Discord webhook with author name, channel, message preview, and running count — every single time.
🗄️
Supabase LoggingEvery detected message is inserted as a row with full metadata. Permanent, queryable, yours.
🛡️
Smart SuppressionTwo-layer system blocks false triggers when you switch channels — URL polling + burst detection.
🧹
DeduplicationTracks message IDs so Discord's optimistic rendering never double-counts your own messages.
📊
Popup DashboardThree-tab UI with live stats, scrollable message log, JSON export, and full reset.

How the pieces connect

Three scripts run independently and talk via Chrome's message passing API:

ScriptRuns inResponsibility
content.jsDiscord tabWatches DOM, detects messages, shows overlay, extracts metadata
background.jsService workerReceives events, fires webhook, writes to Supabase, relays popup commands
popup.jsToolbar popupDisplays stats + log, sends toggle/reset commands, handles export

File structure

feather/ ├── manifest.json — Chrome MV3 manifest, permissions, host rules ├── background.js — Service worker: webhook + Supabase + tab relay ├── content.js — Injected into Discord: observer + overlay ├── popup.html — Popup markup and all inline styles ├── popup.js — Popup logic: tabs, storage, live updates ├── styles.css — Overlay indicator styles injected into Discord └── icons/ ├── icon16.png ├── icon48.png └── icon128.png
Getting Started

Installation

Feather is a private unpacked extension — it never goes through the Chrome Web Store. You load it directly from your files. The process is the same in Chrome, Edge, Brave, and Arc.

ℹ️
Before you startYou need all six files plus the icons/ folder in a single directory on your computer. Configure your credentials in background.js before loading — it's easier than reloading twice.
1
Put all files in one folder
Create a folder somewhere permanent — e.g. C:\Users\you\feather\. Place all six files and the icons\ subfolder inside it. Don't move the folder after loading or the extension will break.
2
Add your credentials to background.js
Open background.js in Notepad, VS Code, or any text editor. Fill in the three constants at the top — your webhook URL, Supabase URL, and Supabase anon key. Save the file. See Configuration for details.
3
Open Chrome Extensions
In Chrome's address bar, type chrome://extensions and press Enter.
4
Enable Developer mode
In the top-right corner of the extensions page, toggle Developer mode on. The page will refresh and show new buttons.
5
Click "Load unpacked"
Click the Load unpacked button that appeared. A file picker opens. Navigate to your feather\ folder and click Select Folder.
6
Pin the extension
The Feather card appears on the extensions page. In the Chrome toolbar, click the 🧩 puzzle-piece icon and pin Feather so you can always reach the popup.
7
Open Discord and verify
Navigate to discord.com/app. Within a couple of seconds the Feather overlay indicator should appear in the bottom-right corner of the page showing watching. Click the toolbar icon to open the popup.

Setup

Configuration

Everything lives in four constants at the very top of background.js. Open the file, fill them in, save, reload the extension. That's it.

// ── background.js — top of file ──────────────────────────

const DISCORD_WEBHOOK_URL = "https://discord.com/api/webhooks/YOUR_ID/YOUR_TOKEN";
const SUPABASE_URL        = "https://your-project-id.supabase.co";
const SUPABASE_ANON_KEY   = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...";
const SUPABASE_TABLE      = "feather_logs"; // change if you renamed your table
⚠️
Reload required after every editGo to chrome://extensions, click the ↺ icon on the Feather card, then reload your Discord tab. Chrome won't pick up changes automatically.

Both integrations are fully optional. If you leave a value as the placeholder string, that integration is silently skipped — no errors thrown. The popup's Overview tab shows live status dots so you can see at a glance what's configured.

Chrome storage keys

Feather stores runtime state in chrome.storage.local. You never manage these directly, but it's useful to know they exist:

feather_enabledboolean
Whether detection is active. Defaults to true on first run.
feather_countnumber
Running session message count. Persists until you clear it.
feather_discord_openboolean
Whether a Discord tab with the content script is currently active.
feather_recentarray
Last 50 detected messages with full metadata. Shown in the Log tab.
feather_integrationsobject
{ webhook: boolean, supabase: boolean } — written by background.js on startup so the popup can show status dots.

You can wipe all of this cleanly using the Full reset button in the popup Settings tab.

manifest.json — why host_permissions matter

Chrome Manifest V3 requires outbound fetch() targets to be explicitly listed in host_permissions. Without this, requests to Supabase and Discord are silently blocked from the service worker — no error, just nothing happens.

"host_permissions": [
  "https://discord.com/*",
  "https://*.supabase.co/*"
]

Both of these are already in the current manifest.json. Don't remove them.

Integration

Webhook

On every detected message, Feather fires a POST to a Discord webhook URL with the author's name, the channel, a message preview, and the running count.

Getting a webhook URL

1
Open a server you manage
You need Manage Webhooks permission on the target channel. This can be any server and any channel — it doesn't have to be the same one you're monitoring. A private logging server works great.
2
Open channel settings
Click the ⚙️ gear icon next to the channel name → Integrations → Webhooks → New Webhook. Give it a name like "Feather".
3
Copy the Webhook URL
Click Copy Webhook URL. It looks like:
https://discord.com/api/webhooks/1234567890/abc123xyz...
4
Paste into background.js
Set DISCORD_WEBHOOK_URL to this value and reload the extension.

What the webhook message looks like

Here's a real example of what gets posted to your logging channel:

# feather-logs
feather webhook Today at 14:32
🔔 New Discord Message · 14:32:07
Author: username123
Channel: general
Message: hey has anyone seen the new update yet
Total count: 7

The message content is capped at 200 characters before sending. Longer messages are truncated with no indication — keep that in mind if you need full content, use Supabase instead.

Testing your webhook

You can test the URL is valid before loading the extension by sending a quick curl from your terminal:

curl -X POST YOUR_WEBHOOK_URL \
  -H "Content-Type: application/json" \
  -d '{"content": "Feather test 🪶"}'

If your logging channel gets a message, the URL is correct.

Integration

Supabase

Every detected message is inserted as a row into your Supabase database — giving you a permanent, queryable log of everything Feather has ever seen.

⚠️
manifest.json must list Supabase in host_permissionsChrome MV3 silently blocks outbound fetches to unlisted hosts from service workers. The current manifest.json already includes https://*.supabase.co/* — do not remove it.

Step 1 — Create a project

Go to supabase.com, sign in, and click New project. Give it a name (e.g. feather), set a database password, pick a region close to you, and wait about 60 seconds for it to provision. The free tier is more than enough — it gives you 500MB and unlimited API calls.

Step 2 — Create the table

In your project sidebar, click SQL Editor. Paste this entire block and click Run:

create table feather_logs (
  id         bigint generated always as identity primary key,
  author     text,
  content    text,
  channel    text,
  count      integer,
  timestamp  timestamptz default now()
);

-- Enable row-level security (required to use the anon key)
alter table feather_logs enable row level security;

-- Allow the anon key to INSERT rows
create policy "Allow anon insert"
  on feather_logs for insert to anon
  with check (true);

-- Allow the anon key to SELECT rows
create policy "Allow anon select"
  on feather_logs for select to anon
  using (true);
ℹ️
Why the RLS policies?Supabase enables Row Level Security by default. Without the insert policy, the anon key (used by the extension) is denied permission to write rows — you'd get a 403 with no helpful error message.

Step 3 — Get your credentials

In your project sidebar go to Project Settings → API. You need two values:

FieldWhereLooks like
Project URLSettings → API → Project URLhttps://abcdefgh.supabase.co
anon / public keySettings → API → Project API keyseyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... (long JWT)

Step 4 — Paste into background.js

const SUPABASE_URL      = "https://abcdefgh.supabase.co";
const SUPABASE_ANON_KEY = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...";
const SUPABASE_TABLE    = "feather_logs";

Reload the extension at chrome://extensions → ↺, then reload Discord.

Step 5 — Verify it's working

Trigger a detection (send a message or have someone else send one in the channel you're watching). Then go to Table Editor → feather_logs in Supabase. A new row should appear within one second.

Useful queries

Run these in the Supabase SQL Editor to explore your data:

-- All messages, newest first
select timestamp, author, channel, content
from feather_logs
order by timestamp desc;

-- Messages per author (leaderboard)
select author, count(*) as total
from feather_logs
group by author
order by total desc;

-- Messages per channel
select channel, count(*) as total
from feather_logs
group by channel
order by total desc;

-- Last 24 hours only
select * from feather_logs
where timestamp > now() - interval '24 hours'
order by timestamp desc;

-- Search for a specific author
select * from feather_logs
where author ilike '%username%'
order by timestamp desc;

Detection Logic

How Feather identifies real messages, suppresses false triggers, deduplicates, and extracts author and channel metadata — all without touching the Discord API.

Identifying a message node

Feather runs a MutationObserver on document.body with { childList: true, subtree: true }. Every added node is checked against three criteria in order:

1
data-list-item-id starts with "chat-messages-"
Discord stamps every message <li> element with this attribute. It's the primary signal and the most reliable.
2
Contains a child with that attribute
Covers wrapper divs Discord occasionally adds around the <li> in certain UI states.
3
Contains a messageContent_ or cozyMessage class
Class-based fallback for older Discord versions or edge cases. Less precise but still reliable.

Channel switch suppression

When you switch channels, Discord bulk-inserts ~50 historical messages into the DOM. Without suppression, each one would trigger a detection. Feather uses two independent layers:

LayerMechanismCooldown
URL pollerChecks location.href every 150ms. Any URL change immediately arms a cooldown.2000ms
Burst detectorCounts message nodes per MutationObserver callback. 3+ nodes in a single callback = history dump → suppressed + cooldown re-armed.2000ms

These run independently. A slow channel switch is caught by the URL poller. A fast bulk insert is caught by the burst detector. Initial page load is suppressed by both (cooldown fires on script init).

If you're in an extremely active channel and legitimate messages arrive in bursts of 3+, you can increase BURST_LIMIT in content.js:

// content.js — near the top
const BURST_LIMIT  = 5;    // raise if false suppressions happen
const NAV_COOLDOWN = 2000; // ms to ignore after URL change

Deduplication

Discord uses optimistic rendering — when you send a message it appears in the DOM immediately (as a "local" message), then gets replaced with the server-confirmed version. Without dedup, this counts as two messages.

Feather maintains a Set of seen data-list-item-id values. If an ID is already in the set, the node is skipped. Nodes with data-is-local-message="true" are also skipped entirely. The set is capped at 500 entries (oldest evicted first) and cleared on channel switch.

Author extraction

Discord groups consecutive messages from the same author — only the first in a group shows the username header. Feather handles this by:

  1. Looking for [class*="username_"] within the message node itself
  2. If not found (grouped message), walking backwards through previous messages in the scroller to find the most recent username header above it
  3. Falling back to "unknown" if neither works

Channel name extraction

The current channel name is read from Discord's page header using:

document.querySelector('h3[class*="title_"]')
  || document.querySelector('[class*="channelName_"]')
  || document.querySelector('h1[class*="title_"]')

This reads whatever channel is currently visible — if you're watching multiple servers simultaneously in separate tabs, each tab tracks its own channel independently.

Help

Troubleshooting

Most issues come down to one of five things. Work through each checklist below before giving up.

Overlay doesn't appear on Discord

🔴
Steps to fix 1. Hard reload Discord: Ctrl Shift R (Windows/Linux) or ⌘ Shift R (Mac).
2. Check that Feather is enabled at chrome://extensions.
3. Open the Discord tab's DevTools (F12) → Console tab → look for any Feather errors.
4. Make sure you're on discord.com not the desktop app — content scripts only inject into the browser.

Popup toggle / reset does nothing

🔴
Steps to fix 1. Go to chrome://extensions → Feather → click the service worker link to open background DevTools.
2. Check the Console tab for any errors. A missing discordTabs variable or unhandled rejection is the usual cause.
3. Click ↺ to reload the extension, then reload the Discord tab — this re-runs FEATHER_CHECKIN and re-registers the tab in the relay.
4. If it still doesn't work, open the Discord tab's DevTools Console and look for message-passing errors.

Webhook messages not arriving

🔴
Checklist ☐ DISCORD_WEBHOOK_URL is set in background.js (not the placeholder).
☐ manifest.json includes "https://discord.com/*" in host_permissions.
☐ Background DevTools Console shows no [Feather] Webhook failed: error.
☐ Test the URL directly with curl (see the Webhook page).

Supabase rows not appearing

🔴
Checklist ☐ SUPABASE_URL and SUPABASE_ANON_KEY are correct (no trailing slashes).
☐ manifest.json includes "https://*.supabase.co/*" in host_permissions.
☐ The feather_logs table exists — re-run the SQL from the Supabase setup page.
☐ Both RLS policies exist (Allow anon insert and Allow anon select).
☐ Background DevTools shows no [Feather] Supabase log failed: error.

Channel switches still trigger false detections

⚠️
Increase the burst limit or cooldown Make sure you're running the latest content.js — earlier versions had no suppression at all. If you still see false triggers, open content.js and increase BURST_LIMIT from 3 to 5, or increase NAV_COOLDOWN from 2000 to 3000. Reload the extension and Discord.

Messages being double-counted

⚠️
Check for duplicate extension loads Go to chrome://extensions — if Feather appears twice, you've loaded it from two different folders. Remove the duplicate. The dedup Set is per-script-instance, so two content scripts in the same tab will each count independently.
