Introduction

Feather docs

A whisper-light Chrome extension that watches Discord for new messages and logs them — with a clean overlay, webhook alerts, and Supabase storage.

👁
Live Detection
MutationObserver watches Discord's DOM for new messages in real time — no polling.
🔔
Webhook Alerts
Fires a Discord webhook with author, channel, and message preview on every detection.
🗄
Supabase Logging
Every message is inserted as a row in your Supabase database with full metadata.
🛡
Smart Suppression
Two-layer channel-switch suppression prevents false triggers from history backfills.

File Structure

feather/ ├── manifest.json — extension manifest (MV3) ├── background.js — service worker, webhook + Supabase ├── content.js — injected into Discord, MutationObserver ├── popup.html — popup markup + inline styles ├── popup.js — popup logic, tabs, live updates ├── styles.css — overlay indicator styles └── icons/ ├── icon16.png ├── icon48.png └── icon128.png

How the pieces talk

The three main scripts communicate via Chrome's message passing API:

MessageDirectionWhat it does
FEATHER_CHECKINcontent → backgroundRegisters the tab so commands can be relayed back
SEND_WEBHOOKcontent → backgroundTriggers webhook POST + Supabase insert
FEATHER_TOGGLEpopup → background → contentPauses or resumes the observer
FEATHER_RESET_COUNTpopup → background → contentResets count to 0, clears dedup history

Getting Started

Installation

Feather is a private unpacked extension — it's not on the Chrome Web Store. Load it directly from your local files.

ℹ️
Chrome or any Chromium-based browser These instructions work for Chrome, Edge, Brave, Arc, and any other Chromium browser. The steps are identical on all platforms once you have the folder.
1
Download the files
Put all six files (manifest.json, background.js, content.js, popup.html, popup.js, styles.css) plus the icons/ folder into a single folder — for example C:\Users\you\feather\.
2
Configure your credentials
Open background.js in Notepad or VS Code and fill in your webhook URL and Supabase details at the top. See the page.
3
Open Chrome extensions
In Chrome, navigate to chrome://extensions in the address bar.
4
Enable Developer mode
Flip the Developer mode toggle in the top-right corner of the extensions page.
5
Load unpacked
Click Load unpacked, then navigate to and select your feather\ folder. The Feather icon appears in your toolbar.
6
Open Discord
Navigate to discord.com/app. The overlay indicator appears in the bottom-right corner.

Setup

Configuration

All configuration is done by editing the three constants at the very top of background.js. There is no settings UI — it's intentionally hardcoded for a private extension.

// ── background.js — top of file ───────────────

const DISCORD_WEBHOOK_URL = "https://discord.com/api/webhooks/...";
const SUPABASE_URL        = "https://your-project.supabase.co";
const SUPABASE_ANON_KEY   = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...";
const SUPABASE_TABLE      = "feather_logs"; // your table name

Both integrations are optional. If you leave a value as the placeholder string, that integration is silently skipped — no errors. The popup Overview tab shows live status dots indicating whether each is configured.

⚠️
After every edit to background.js You must reload the extension at chrome://extensions (click the ↺ icon on the Feather card) and then reload your Discord tab.

Chrome Storage

Runtime state is stored in chrome.storage.local. You don't manage these directly, but it's useful to know what exists:

KeyTypeDescription
feather_enabledbooleanWhether detection is active. Defaults to true.
feather_countnumberRunning session message count.
feather_discord_openbooleanWhether a Discord tab with the content script is live.
feather_recentarrayLast 50 detected messages with full metadata.
feather_integrationsobject{ webhook, supabase } booleans — set by background.js on startup.

You can wipe all of this cleanly using the Full reset button in the popup's Settings tab.

Integration

Webhook

On every detected message, Feather sends a POST request to a Discord webhook URL with author, channel, message preview, and running count.

Getting a webhook URL

1
Open a Discord server you control
You need Manage Webhooks permission on the channel. It can be any server and any channel — it doesn't have to be the one you're monitoring.
2
Go to channel settings
Click the gear icon next to the channel name → Integrations → Webhooks → New Webhook.
3
Copy the URL
Click Copy Webhook URL. It looks like https://discord.com/api/webhooks/123.../abc...
4
Paste into background.js
Set DISCORD_WEBHOOK_URL to this value, reload the extension.

Payload format

Each webhook POST sends a JSON body with a single content field:

🔔 New Discord Message · 14:32:07
Author:  username
Channel: general
Message: hey whats up everyone
Total count: 12
💡
Message content is capped at 200 characters Longer messages are truncated before sending to keep webhook payloads small.

Integration

Supabase

Every detected message is inserted as a row into your Supabase database — giving you a permanent, queryable log of everything Feather has seen.

⚠️
manifest.json must include host_permissions Chrome MV3 silently blocks outbound fetches to hosts not listed in host_permissions. The current manifest already includes https://*.supabase.co/* — do not remove it.

1. Create a Supabase project

Go to supabase.com, create a new project, and wait about a minute for it to provision. The free tier is more than enough.

2. Create the table

In your project, go to SQL Editor and run this entire block:

create table feather_logs (
  id         bigint generated always as identity primary key,
  author     text,
  content    text,
  channel    text,
  count      integer,
  timestamp  timestamptz default now()
);

-- Allow the anon key to insert and read rows
alter table feather_logs enable row level security;

create policy "Allow anon insert"
  on feather_logs for insert to anon
  with check (true);

create policy "Allow anon select"
  on feather_logs for select to anon
  using (true);

3. Get your credentials

Go to Project Settings → API and copy two values:

ValueWhere to find itLooks like
Project URLSettings → API → Project URLhttps://abcdef.supabase.co
anon / public keySettings → API → Project API keyseyJhbGci... (long JWT)

4. Paste into background.js

const SUPABASE_URL      = "https://abcdef.supabase.co";
const SUPABASE_ANON_KEY = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...";
const SUPABASE_TABLE    = "feather_logs";

5. Verify it works

After reloading the extension and triggering a detection in Discord, go to Table Editor → feather_logs in Supabase. A new row should appear within a second.

Useful queries

-- All messages, newest first
select timestamp, author, channel, content
from feather_logs
order by timestamp desc;

-- Messages per author
select author, count(*) as total
from feather_logs
group by author
order by total desc;

-- Last 24 hours only
select * from feather_logs
where timestamp > now() - interval '24 hours'
order by timestamp desc;

Detection Logic

How Feather identifies real messages, extracts metadata, and avoids false positives.

Identifying a message node

A DOM node is classified as a message if any of these are true — checked in order:

1
data-list-item-id starts with "chat-messages-"
Discord stamps every message <li> with this attribute. It's the primary, most reliable signal.
2
Contains a child with that attribute
Covers wrapper divs that Discord occasionally adds around the <li>.
3
Contains a messageContent_ or cozyMessage element
Class-based fallback for older Discord versions or edge cases.

Channel switch suppression

When you switch channels Discord bulk-inserts ~50 historical messages into the DOM at once. Feather uses two independent layers to prevent these from triggering detections:

LayerHow it worksWhat it catches
URL pollerChecks location.href every 150ms. Any change arms a 2000ms cooldown.Every channel switch, regardless of how Discord navigates internally.
Burst detectorCounts message nodes per MutationObserver callback. 3+ nodes = history dump → suppressed.Fast bulk inserts that happen before the URL poller fires.

Deduplication

Discord uses optimistic rendering — when you send a message it appears in the DOM immediately, then gets replaced when the server confirms it. Without deduplication this counts as two messages.

Feather tracks seen data-list-item-id values in a Set. Any ID already in the set is skipped. Nodes with data-is-local-message="true" (your own outgoing messages before confirmation) are also skipped. The set is capped at 500 entries.

Author extraction

Discord groups consecutive messages from the same author — only the first shows the username. Feather first checks the message node itself for a [class*="username_"] element. If not found (grouped message), it walks backwards through previous messages in the scroller to find the most recent header above it.

Help

Troubleshooting

Common issues and how to fix them.

Overlay doesn't appear on Discord

🔴
Fix: Hard reload Discord with Ctrl+Shift+R (or ⌘+Shift+R on Mac). If still missing, go to chrome://extensions, check Feather is enabled, click ↺, then reload Discord.

Toggle / reset buttons do nothing

This usually means the background service worker crashed or the discordTabs relay is broken. To debug:

1
Open background DevTools
Go to chrome://extensions → Feather → click the service worker link to open DevTools for the background script.
2
Check the Console tab
Any [Feather] error messages will appear here. A missing discordTabs or unhandled promise rejection is the usual culprit.
3
Reload everything
Click ↺ on the Feather card, then reload the Discord tab. This re-runs FEATHER_CHECKIN and re-registers the tab.

Webhook messages not sending

🔴
Checklist: (1) DISCORD_WEBHOOK_URL is set in background.js and is not the placeholder. (2) manifest.json includes "https://discord.com/*" in host_permissions. (3) Check the background DevTools Console for [Feather] Webhook failed:.

Supabase rows not appearing

🔴
Checklist: (1) SUPABASE_URL and SUPABASE_ANON_KEY are set correctly. (2) manifest.json includes "https://*.supabase.co/*" in host_permissions. (3) The RLS insert policy exists — re-run the SQL from the . (4) Check for [Feather] Supabase log failed: in background DevTools.

Channel switches still trigger false detections

If the burst limit of 3 is too low for a very active channel, increase it in content.js:

// content.js — line ~25
const BURST_LIMIT = 5; // increase from 3
const NAV_COOLDOWN = 2500; // or increase cooldown (ms)

Then reload the extension.

Double-counting messages

This is prevented by the seenMessageIds dedup set. If it's happening, the most likely cause is that content.js is being injected twice (e.g. you have two versions of Feather loaded). Check chrome://extensions for duplicates.
